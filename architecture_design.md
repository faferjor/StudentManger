# 大学生小组课题选题管理系统 - C/S架构设计方案

## 一、整体架构设计

### 1.1 系统架构图
```
┌─────────────────┐    TCP/IP网络通信    ┌─────────────────┐
│   Qt客户端      │◄────────────────────►│   Qt服务器端    │
│                 │                      │                 │
│ • 用户界面      │                      │ • 网络通信模块  │
│ • 业务逻辑处理  │                      │ • 请求处理引擎  │
│ • 网络请求封装  │                      │ • 数据访问层    │
│ • 本地缓存管理  │                      │ • 权限验证中心  │
└─────────────────┘                      └─────────────────┘
                                                   │
                                                   ▼
┌─────────────────┐                      ┌─────────────────┐
│   用户群体      │                      │   MySQL数据库   │
│ • 系统管理员   │                      │ • 用户数据表    │
│ • 指导教师     │                      │ • 课题数据表    │
│ • 学生小组     │                      │ • 申请数据表    │
└─────────────────┘                      └─────────────────┘
```

### 1.2 架构优势
1. **数据安全**：数据库仅在服务器端暴露，客户端无法直接访问数据库
2. **集中管理**：所有业务逻辑和数据集中在服务器，便于维护和升级
3. **性能优化**：服务器端可以进行数据缓存和查询优化
4. **扩展性强**：支持多客户端并发连接，便于系统扩容
5. **权限控制**：服务器端统一进行权限验证和请求过滤

## 二、通信协议设计

### 2.1 数据传输格式
采用JSON格式进行数据传输，便于解析和扩展

```json
{
    "request_id": "唯一请求ID",
    "command": "请求命令字",
    "user_info": {
        "user_id": 1,
        "username": "admin",
        "user_type": 1
    },
    "data": {
        // 请求数据内容
    },
    "timestamp": 1779343703
}
```

### 2.2 响应格式
```json
{
    "request_id": "对应请求ID",
    "status": "success/fail",
    "message": "响应消息",
    "data": {
        // 响应数据内容
    },
    "timestamp": 1779343703
}
```

### 2.3 核心命令字
| 命令字 | 功能描述 | 用户权限 |
|--------|----------|----------|
| USER_LOGIN | 用户登录 | 所有用户 |
| USER_LOGOUT | 用户退出 | 所有用户 |
| USER_QUERY | 查询用户信息 | 管理员/教师 |
| USER_ADD | 添加用户 | 管理员 |
| USER_UPDATE | 更新用户信息 | 管理员/本人 |
| USER_DELETE | 删除用户 | 管理员 |
| TOPIC_QUERY | 查询课题信息 | 所有用户 |
| TOPIC_ADD | 发布课题 | 管理员 |
| TOPIC_UPDATE | 更新课题信息 | 管理员/指导教师 |
| TOPIC_DELETE | 删除课题 | 管理员 |
| APPLY_SUBMIT | 提交选题申请 | 学生小组 |
| APPLY_QUERY | 查询申请状态 | 所有用户 |
| APPLY_REVIEW | 审核选题申请 | 教师/管理员 |
| STATISTICS_QUERY | 查询统计数据 | 管理员/教师 |

## 三、服务器端设计

### 3.1 核心模块

#### 3.1.1 网络通信模块
- 基于QTcpServer实现多客户端并发连接
- 每个客户端连接对应一个独立的处理线程
- 实现数据的接收、解析和响应发送
- 处理客户端连接超时和异常断开

#### 3.1.2 请求处理引擎
- 负责解析客户端请求命令
- 根据命令字分发到对应的业务处理模块
- 实现请求的异步处理和响应
- 记录请求日志和性能统计

#### 3.1.3 数据访问层
- 封装数据库操作接口
- 实现数据的增删改查操作
- 处理数据库连接池和事务
- 实现数据缓存和查询优化

#### 3.1.4 权限验证中心
- 验证用户登录状态和权限
- 实现基于角色的访问控制(RBAC)
- 过滤非法请求和越权操作
- 记录安全审计日志

## 八、部署方案

### 8.1 开发环境部署
```
1. 安装Qt 6.5.3及以上版本
2. 安装MySQL 8.0及以上版本
3. 创建数据库并导入初始化数据
4. 编译并启动服务器端
5. 编译并启动客户端进行测试
```

### 8.2 生产环境部署

