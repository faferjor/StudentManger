# 大学生小组课题选题管理系统

基于Qt 6.5.3 + C/S架构的大学生小组课题选题管理系统，支持管理员、教师、学生小组三类用户角色，实现课题发布、选题申请、审核管理等完整业务流程。

## 一、系统架构

### 1.1 整体架构
```
┌─────────────────┐    TCP/IP网络通信    ┌─────────────────┐
│   Qt客户端      │◄────────────────────►│   Qt服务器端    │
│                 │                      │                 │
│ • 用户界面      │                      │ • 网络通信模块  │
│ • 业务逻辑处理  │                      │ • 请求处理引擎  │
│ • 网络请求封装  │                      │ • 数据访问层    │
│ • 本地缓存管理  │                      │ • 权限验证中心  │
└─────────────────┘                      └─────────────────┘
                                                   │
                                                   ▼
┌─────────────────┐                      ┌─────────────────┐
│   用户群体      │                      │   MySQL数据库   │
│ • 系统管理员   │                      │ • 用户数据表    │
│ • 指导教师     │                      │ • 课题数据表    │
│ • 学生小组     │                      │ • 申请数据表    │
└─────────────────┘                      └─────────────────┘
```

### 1.2 技术栈
- **客户端**：Qt 6.5.3 (C++ Widgets)
- **服务器端**：Qt 6.5.3 (C++ Network)
- **数据库**：MySQL 8.0+
- **通信协议**：TCP/IP + JSON
- **开发环境**：Qt Creator 11.0+

## 二、核心功能

### 2.1 用户角色分类

#### 1. 系统管理员
- 用户管理（增删改查管理员、教师、学生小组账号）
- 课题管理（发布、编辑、删除课题，设置选题人数限制）
- 申请管理（查看所有选题申请，最终审核确认）
- 数据统计（全局选题情况统计，数据导出）
- 系统配置（系统参数设置，数据备份）

#### 2. 指导教师
- 个人课题管理（查看负责的课题，编辑课题信息）
- 申请审核（查看申请自己课题的学生小组，进行审核）
- 学生管理（查看选择自己课题的学生小组信息）
- 统计查看（个人课题选题情况统计）

#### 3. 学生小组
- 课题浏览（查看所有可选课题，支持搜索筛选）
- 选题申请（提交课题申请，查看申请状态）
- 申请管理（撤销未审核的申请，查看驳回原因）
- 个人信息（完善小组信息，修改成员信息）

### 2.2 核心业务流程

#### 课题发布流程
```
管理员 → 发布课题 → 填写课题信息 → 保存到数据库 → 学生可见
```

#### 选题申请流程
```
学生 → 浏览课题 → 选择课题 → 提交申请 → 教师审核 → 结果通知
```

#### 申请审核流程
```
教师 → 查看申请 → 审核（通过/驳回） → 更新状态 → 通知学生
```

## 四、环境搭建

### 4.1 开发环境要求
- Qt 6.5.3 或更高版本
- MySQL 8.0 或更高版本
- CMake 3.16 或更高版本
- 支持C++17的编译器（MSVC 2019+，GCC 8+，Clang 9+）

### 4.2 数据库配置
1. 创建MySQL数据库：
```sql
CREATE DATABASE student_topic_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 导入初始化数据：
```bash
mysql -u root -p student_topic_system < database.sql
```

3. 修改服务器配置文件（`server/config/database_config.ini`）：
```ini
[Database]
HostName=localhost
DatabaseName=student_topic_system
UserName=root
Password=your_password
Port=3306
```

### 4.3 编译运行

#### 编译服务器端
```bash
cd server
mkdir build && cd build
cmake ..
cmake --build . --config Release
```

#### 编译客户端
```bash
cd client
mkdir build && cd build
cmake ..
cmake --build . --config Release
```

#### 启动服务
1. 启动服务器：
```bash
./server/build/StudentTopicSystemServer
```

2. 启动客户端：
```bash
./client/build/StudentTopicSystemClient
```

## 五、使用说明

### 5.1 默认账号
- 管理员：`admin` / `admin123`
- 教师账号：需管理员创建
- 学生小组账号：需管理员创建

### 5.2 系统操作

#### 管理员操作流程
1. 登录系统 → 用户管理 → 创建教师和学生小组账号
2. 课题管理 → 发布课题 → 填写课题信息
3. 申请管理 → 查看申请 → 最终审核确认
4. 数据统计 → 查看全局选题情况

#### 教师操作流程
1. 登录系统 → 我的课题 → 查看负责的课题
2. 申请审核 → 处理学生小组的申请
3. 学生管理 → 查看选择自己课题的学生小组

#### 学生小组操作流程
1. 登录系统 → 完善小组信息
2. 课题浏览 → 搜索筛选感兴趣的课题
3. 提交申请 → 等待教师审核
4. 查看状态 → 根据审核结果进行后续操作

## 六、系统特性

### 6.1 安全特性
- **数据加密**：密码MD5加密存储，网络传输可扩展SSL加密
- **权限控制**：基于角色的访问控制，严格的权限验证
- **防攻击**：登录失败次数限制，请求超时处理
- **数据备份**：支持数据导出和备份功能

### 6.2 性能特性
- **并发处理**：服务器端多线程并发处理，支持多客户端连接
- **缓存机制**：本地数据缓存，减少网络请求
- **异步加载**：界面异步加载，提升用户体验
- **查询优化**：数据库索引优化，提高查询效率

### 6.3 用户体验
- **友好界面**：现代化UI设计，操作简单直观
- **响应式设计**：支持不同分辨率的显示设备
- **实时提示**：操作结果实时反馈，错误提示清晰
- **多语言支持**：支持中英文界面切换（可扩展）
